git常用指令笔记

发表于 2021-10-30 本文字数： 1.8k 阅读时长 ≈ 6 分钟

git全局配置

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

git初始化

$ git init

git添加到暂存区(stage\index)

$ git add file	  // file填写你自己的文件名

git一键提交到版本库

$ git commit -m "此内容为提交的说明"   // 此命令可以一键提交很多文件

git状态查看

$ git status

git查看文件修改前后的不同

$ git diff file		// file填写你自己修改了的文件名

git查看提交到版本库的历史记录

$ git log		// 记录包括版本号、作者、时间、提交说明
$ git log --pretty=oneline	// 一行显示版本号和提交说明

git回退版本库

$ git reset --hard HEAD^  // 回退到上一个版本，HEAD^^上上个版本
$ git reset --hard HEAD~1   // 回退到上一个版本，数字代表回退到上几个版本号

git回到未来某个版本

当执行了上述回退版本库的命令后，如果想再次回到原先的版本库。可以执行下面命令。

$ git reset --hard 版本号	 // 输入想回去的原先的版本号，输入前几位就行了。

git记录每次执行的命令

$ git reflog
e475afc HEAD@{1}: reset: moving to HEAD^
1094adb (HEAD -> master) HEAD@{2}: commit: append GPL
e475afc HEAD@{3}: commit: add distributed
eaadf4e HEAD@{4}: commit (initial): wrote a readme file

git版本库结构体

git查看工作区和版本库里文件的区别

前提是工作区文件和版本库文件是不同的。

$ git diff HEAD -- file		// file填写你自己的文件名

git丢弃工作区的修改(没有提交到暂存区)

$ git checkout -- file 		// file填写你自己的文件名
$ git restore file			// file填写你自己的文件名

git丢弃暂存区的文件(重新放回工作区)

$ git restore --staged file    	// file填写你自己的文件名
$ git reset HEAD file			// file填写你自己的文件名

git删除文件

分为以下两个情况：

  1. 在工作区删除文件时：
$ rm test			// test要删除的文件，你自己可以修改
$ git status
On branch master
Changes not staged for commit:
  (use "git add/rm <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        deleted:    test		// 这里deleted:    test   应该是红色的

上面情况就相当于修改了工作区里的文件，然后可以进行add提交或者restore恢复。

  1. 在工作区提交删除文件的操作时
$ git rm test		// test要删除的文件，你自己可以修改
$ git status
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        deleted:    test		// 这里deleted:    test   应该是绿色的

上面的 $ git rm test 命令实际上执行了两步，删除文件和提交文件到暂存区。所以当你执行命令 $ git restore –staged test时，会回到第(1)种情况的status。

3.恢复到最新版本

$ git checkout -- test

git checkout其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。

ps：但是当工作区的文件提交到暂存区的时候(也就是当属于第(2)种情况的时候)，用该命令会报错。

获取SSH密钥

  1. 首先在用户主目录查看有没有.ssh目录，也就是C:\user\home\.ssh(这里换成你自己的路径)。
  2. 没有的话，打开Shell(Windows下打开Git Bash)，创建SSH Key:
$ ssh-keygen -t rsa -C "youremail@example.com"	
			// 引号里填写你之前设置的全局配置的邮箱

  1. 一路回车、yes即可。

  2. 在用户主目录找到.ssh目录。里面有两个文件。id_rsa和id_rsa.pub。这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

  3. 进入Github，登录账户，进入Settings里，找到SSH，填写任意title，在key文本框里粘贴id_rsa.pub内容

  4. 你可以通过Shell(Windows下打开Git Bash)，进入到.ssh目录，然后输入 $ cat id_rsa.pub获取内容

本地仓库关联到远程库

$ git remote add origin git@github.com:wuxin-123/learngit.git

wuxin-123换成你自己的账户名，origin是远程库名

git推送远程库

$ git push origin master	// 把本地master分支的版本库的内容推送到远程库里

查看远程库信息

$ git remote -v

删除远程库(实际上是取消本地库与远程库的关联)

$ git remote rm origin

克隆远程库

git支持多种协议，下面换成https协议的URL也是可以的。

// wuxin-123改成你自己的Github账户名

$ git clone git@github.com:wuxin-123/gitskills.git

git新建分支

$ git checkout -b dev	// 创建dev分支并切换到dev分支。(不推荐使用)
相当于
$ git branch dev		// 创建dev分支
$ git checkout dev		// 切换到dev分支。(不推荐)

$ git switch -c dev		// 创建dev分支并切换到dev分支。(推荐)
$ git switch dev		// 切换到dev分支。(推荐)

git查看分支

$ git branch

git合并分支

$ git merge dev  // (快速合并)把指定分支合并到当前分支，即把dev分支合并到你现在的分支
$ git merge --no-ff -m "merge with no-ff" dev 
				 // 不使用快速合并模式，并且创建一个新commit

git删除分支

$ git branch -d dev    // 如果已经master已经合并了dev分支，不报错，否则，报错
$ git branch -D dev	   // 无论有没有合并，强制删除dev分支

git查看日志

$ git log --graph --pretty=oneline --abbrev-commit

–graph可以查看提交历史记录图，pretty=oneline之前介绍过，就是一行输出。abbrev-commit就是输出版本号的前几位。

git隐藏当前工作区和暂存区

$ git stash

查看隐藏的工作区和暂存区

$ git stash list

恢复原先的工作现场

$ git stash apply 	// 恢复后，stash内容并不删除，即list里的内容还在
$ git stash drop 	// 删除list内容的stash@{0}
$ git stash pop		// 恢复的同时，把stash内容也删除。

git从远程库抓取

$ git pull	  // 从远程库中把最新的提交抓取到本地仓库里，并合并在本地分支。
			  // 报错了，就按照提示进行本地分支和远程分支链接。

新建标签

默认标签是打在最新提交的commit上的

$ git tag v1.0		// 新建V1.0标签，默认打在最近commit上的
$ git tag v0.9 f52c633  	// 新建V0.9标签，打在版本号为f52c633的commit上

查看标签详细信息

$ git show V0.9		// V0.9可更改

创建带有说明的标签

$ git tag v0.1 -m "version 0.1 released" 1094adb	
			// 新建V0.1标签，并进行说明，打在1094版本号

注意：标签总是和某个commit挂钩。如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签。

删除标签

$ git tag -d v0.9		// 删除本地标签
$ git push origin :refs/tags/v0.9		// 删除远程库标签

推送标签

$ git push origin v1.0			
$ git push origin --tags		// 一次性推送所有的本地标签到远程库
