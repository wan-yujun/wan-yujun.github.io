分为以下两个情况：

  1. 在工作区删除文件时：

上面情况就相当于修改了工作区里的文件，然后可以进行add提交或者restore恢复。

  1. 在工作区提交删除文件的操作时

上面的 $ git rm test 命令实际上执行了两步，删除文件和提交文件到暂存区。所以当你执行命令 $ git restore –staged test时，会回到第(1)种情况的status。

3.恢复到最新版本

git checkout其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。

ps：但是当工作区的文件提交到暂存区的时候(也就是当属于第(2)种情况的时候)，用该命令会报错。

获取SSH密钥

  1. 首先在用户主目录查看有没有.ssh目录，也就是C:\user\home\.ssh(这里换成你自己的路径)。
  2. 没有的话，打开Shell(Windows下打开Git Bash)，创建SSH Key:

  1. 一路回车、yes即可。

  2. 在用户主目录找到.ssh目录。里面有两个文件。id_rsa和id_rsa.pub。这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

  3. 进入Github，登录账户，进入Settings里，找到SSH，填写任意title，在key文本框里粘贴id_rsa.pub内容

  4. 你可以通过Shell(Windows下打开Git Bash)，进入到.ssh目录，然后输入 $ cat id_rsa.pub获取内容
